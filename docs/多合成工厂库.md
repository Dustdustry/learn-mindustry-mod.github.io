
# 像素工厂 Mindustry MultiCrafterLib 多配方工厂制作教程

经过我不懈的努力，我终于把lplum的多合成工厂前置模组给研究明白了。下面我将分享具体操作方法，希望对你有帮助。

## 一、前置准备

### 1. 所需文件

- 核心依赖库：`MultiCrafterLib-2.0.0.jar`加入教程QQ群:[L.M.M.无限责任天堂](http://qm.qq.com/cgi-bin/qm/qr?_wv=1027&k=LGT1dGUqX1TAmB00S4AGigWcsjr9yGzY&authKey=w7s9Qr22gRI%2FjkKRPw9yViDZu%2BGaAZwQQ2Fh8DrS2ZiydVYz9aZLS%2FYaW7kL%2FieX&noverify=0&group_code=497913479)以获取

- JS扩展文件：`multicCrafterlib.js`（**仅使用JS编写时需要**，纯JSON配置无需此文件）

因为这个文件内容不长，所以我就直接放在这里,需要的复制下来就可以了

```js

const loader = Vars.mods.mainLoader();
const scripts = Vars.mods.scripts;
const NativeJavaClass = Packages.rhino.NativeJavaClass;
function getClass(name){
  return NativeJavaClass(scripts.scope, loader.loadClass(name));
};

module.exports = {
    MultiCrafter : getClass("multicraft.MultiCrafter"),
    DrawRecipe : getClass("multicraft.DrawRecipe"),
    IOEntry : getClass("multicraft.IOEntry"),
    Recipe : getClass("multicraft.Recipe")
}

print(">>>>>MultiCrafter JavaScript loaded.")

```

## 二、基础安装与配置

### 步骤1：文件部署

1.解压 `MultiCrafterLib-2.0.0.jar`，将`multicraft/`，`net/`和`classes.dex` 文件放入模组根目录。

2.若使用JS开发，额外将 `multicCrafterlib.js` 放入模组的 `scripts` 目录。

### 步骤2：`mod.json` 核心配置

在你的模组 `mod.json` 文件中，添加Java主类声明，完整示例如下：

```json

{
  "name": "你的模组名",
  "displayName": "模组显示名",
  "author": "作者名",
  "version": "1.0",
  "java": true,
  "main": "net.lplum.MultiCrafterMod",
  // 其他原有配置保持不变
}

```

## 三、工厂类型定义

### 1. JSON 配置方式

如果使用JSON定义工厂，直接在工厂的 `type` 字段填写：

```json
"type": "multicraft.MultiCrafter"
```

无需额外JS文件，直接通过JSON即可配置所有配方，零代码上手。

## 四、核心：配方(`recipes`)配置

以下是一个正确的配方(`recipes`)配置案例

```json

"recipes": [
  {
    "input": {
      "items": ["copper/2", "lead/1"],
      "fluids": ["water/1.0"],
      "heat": 2,
      "power": 5,
    },
    "output": {
      "items": ["titanium/1"],
      "fluids": ["slag/0.5"]
    },
    "craftTime": 60,
    
    
  },
  {
    "input": {
      "items": ["titanium/1", "silicon/2"],
      "fluids": ["oil/0.8"]
    },
    "output": {
      "items": ["surge-alloy/1"],
      "fluids": ["oil/0.8"],
      "heat": 2,
      "power": 5,
    },
    "craftTime": 120,
    "craftEffect": "smoke"
  }
]
```

每个配方也可单独配置 `icon`、`iconColor` 和 `craftEffect`，实现不同配方的差异化UI展示。

> 注：JSON可用的icon范围有限，JS可通过 `prov(() => Core.atlas.find("liquid-neoplasm"))` 来使用任意贴图以更丰富的自定义绘制，满足复杂界面需求。

其配置逻辑和原生工厂大概相同，但部分接口有关键区别，务必注意以下规则：

- **液体字段**：原生工厂用 `liquids`，本库统一使用 `fluids`，格式为：

  ```json
  "fluids": [
    "water/1.2", "oil/0.8"
  ]
  ```

  数组内格式为 `["液体1/数量","液体2/数量"]`
- **制作时间**：**无全局统一 `craftTime`**，每个配方都有独立的制作时间，必须在单个配方内单独配置
- **输入输出**：物品输入输出格式和原生完全一致，支持多物品、多液体同时配置

## 五、多配方分别绘制("multicraft.DrawRecipe")

如果你希望不同的配方有不同的Draw，你可以使用`multicraft.DrawRecipe`，下面是参考案例:

```json
"drawer": {
    "type": "multicraft.DrawRecipe"
    "defaultDrawer": 0, //在建筑界面显示的图标
    "drawers": [
        //对应配方一
        {
            //这里面的就和你平常写的相同了
            "type": "DrawMulti"
            "drawers": [
                {
                    "type": "DrawDefault"
                }
                {
                    "type": "DrawFlame"
                    "color": "ffef99"
                }
            ]
        },
        //对应配方二
        {
            "type": "DrawMulti"
            "drawers": [
                {
                    "type": "DrawDefault"
                }
                {
                    "type": "DrawFlame"
                    "color": "ffef99"
                }
            ]
        }
        
    ]
}
```json

> 注意defaultDrawer:0是第一个配方的Draw,defaultDrawer:1是第二个配方的Draw。

## 六、UI与界面扩展（`menu`接口）
工厂额外提供 `menu` 接口，用于自定义UI样式，支持4种预设风格：
| 风格类型 | 效果说明 |
|----------|----------|
| `transform` | 带变形动画的交互风格 |
| `simple` | 极简展示风格 |
| `number` | 数字进度展示风格 |
| `detailed` | 详细信息展示风格 |

### 配置示例

```json
"menu": "detailed",
```
